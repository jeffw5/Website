Relation Taxonomy SKOS for vocabularies/taxonomies · OWL/RDFS for ontologies

Relations are grouped into families. Each family declares which model kind it applies to: vocab SKOS for vocabularies & taxonomies, ontology OWL/RDFS for ontologies, or any cross-cutting (e.g. Dublin Core). The Model Editor offers these predicates when you add a relation and softly warns if a predicate's family doesn't match the node's model kind. Changes save to this browser and the editor picks them up live.
